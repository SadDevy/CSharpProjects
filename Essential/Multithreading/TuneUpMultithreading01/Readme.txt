1. Средства синхронизации, которые используются в проекте:
  Класс ReaderWriterLockSlim - Представляет блокировку, используемую для управления доступом к ресурсу, которая позволяет нескольким потокам производить считывание или получать монопольный доступ на запись.

2. Общими данными для потоков является свойство SharedRes класса Resulter.

3. Сообщения System.Diagnostics.Debug будут видны только в режиме Debug сборки.
Можно отлаживать приложение в Debug сборке, чтобы получаться все выходные данные отладки,
а когда приложение будет готово к выпуску, можно скомпилировать сборку Release, не включая
сообщения System.Diagnostics.Debug.
  Сообщения System.Diagnostics.Trace видны как в режиме Debug, так и в режиме Release
сборки.  Они позволяют отслеживать статус приложения в производственной среде. Это особенно
важно в бизнес-приложении, которое использует несколько компонентов, работающих на нескольких
компьютерах.
